# 管理端前端坑:缓存 / 路由 / 组件

> 本页收录管理端网页(Vue3 + Vite 单页应用)这些年踩过的前端坑,按缓存、路由与组件状态、样式、组件细节、兼容与工程五类整理,每条都给到可以直接抄走的铁律。给前端工程师、也给排查「用户说页面不对」的所有人看。

**读完你会知道:**

- 发版之后用户报「新功能没上」「bug 修了还在」,第一句话该问什么
- 单页应用切页面时状态莫名丢失的两种典型病灶,以及 `router-view` 的 key 到底该绑什么
- 暗色模式、全局工具类、日期组件这些「小事」各有哪些一脚踩空的姿势
- 老安卓设备白屏和长期分叉分支合并的正确姿势

前端坑有个共同特点:**症状离根因很远**。用户看到的是「查询报错」「页面是旧的」「筛选丢了」,根因却藏在缓存策略、路由 key、CSS 作用域这些平时不看第二眼的地方。所以这一页每条都严格按「症状 → 根因 → 铁律」写——先认症状,再讲病灶,最后给一条以后不再踩的规矩。

## 一、缓存:发版后疑难杂症的最大来源

发版之后收到的「奇怪反馈」,十有八九不是代码问题,是缓存问题。缓存分两层:用户浏览器这一层,和 CDN 边缘节点那一层,两层都咬过我们。

### 1. 旧缓存跑旧 JS

- **症状**:用户报的 bug 你死活复现不了;或者新功能明明发了,用户坚持说「没上」。你打开是好的,他打开就不对。
- **根因**:用户的浏览器(或 PWA 外壳)缓存了旧构建的 JS,页面壳是新的、逻辑是旧的,甚至整个应用都是旧的。我们有一次「订单导出一片空白」查了半天,最后发现就是 PWA 缓存跑着旧 JS。
- **铁律**:发版后收到疑难杂症,**第一句话让用户强刷(Ctrl+Shift+R)**,强刷好了就不用查代码。工程上治本:静态资源文件名带内容 hash(Vite 默认就有,别关),`index.html` 设短缓存或不缓存——只要 html 是新的,它引用的 hash 文件名自然指向新资源。

### 2. CDN 边缘缓存

- **症状**:发了版,你自己强刷也确认新版本上了,但仍有一部分用户看到旧页面,而且按地域/运营商分布,不是全员。
- **根因**:`index.html` 被 CDN 边缘节点缓存了。上一条治的是浏览器,这一条是边缘节点又把「入口 html」存了一份——html 是旧的,引用的还是旧 hash 的资源,浏览器强刷也没用,因为拉回来的 html 本身就是旧的。
- **铁律**:`index.html` 在 CDN 侧只允许**60 秒级**的短缓存,发版流程里加一步**主动 purge 边缘缓存**;`/api` 路径必须显式排除在 CDN 缓存规则之外——接口被边缘缓存一次,你会得到一批「数据不更新」的灵异工单。

这两条合起来是一条完整的链:**浏览器缓存 → CDN 缓存 → 源站**,发版排障时从近到远逐层问「这一层拿到的是新的吗」。

## 二、路由与组件状态:切个页面,状态没了

### 3. 顶层路由重挂

- **症状**:从列表页切到详情页再切回来,筛选条件全丢了;或者切换页面时整个界面「闪一下」,侧边栏、头部全部重来一遍。
- **根因**:顶层 `router-view` 的 `key` 绑了完整路径(`$route.fullPath` 一类)。任何路由参数变化都会让 key 变化,Vue 认为是全新组件,把**布局组件连同里面所有子页面状态**整个销毁重建。
- **铁律**:顶层 `router-view` 的 key 用**匹配到的第一级路由**(`$route.matched[0]` 的 path),不用 fullPath。这样只有跨一级模块才重建布局,模块内部跳转布局保持存活。以后遇到「切页丢状态」,先验证组件是不是被重建了(生命周期钩子里打个 log 就知道),再谈别的。

### 4. keep-alive 回来不刷新

- **症状**:列表页跳去编辑页改完数据,切回列表,显示的还是改之前的旧数据。
- **根因**:列表页被 `keep-alive` 缓存了——这本来是为了保住筛选和滚动位置,副作用是切回来不走 `onMounted`,不会重新请求。
- **铁律**:被 keep-alive 的列表页在 `onActivated` 里刷新数据;而且**刷新时页码要归位到第一页**——不归位的话,用户在第 5 页操作完回来,数据总量变了,第 5 页可能已经不存在,又是一个新 bug。

这两条(3 和 4)是一对镜像:一个是「不想重建的被重建了」,一个是「被缓存住了但没跟着刷新」。判断方向就一句话:先搞清楚组件到底重建没有,再决定往哪边修。

### 5. 表单离开误弹窗

- **症状**:用户打开一个编辑页,什么都没碰,点返回却弹「有未保存的修改,确定离开?」——狼来了喊多了,真有修改时用户也直接点确定了。
- **根因**:脏检查的「基线快照」拍早了。表单数据是异步回填的:组件挂载时先拍了一张空表单的快照,数据回填后表单自然和空快照不一致,于是永远「脏」。
- **铁律**:基线快照必须在**数据加载完成之后**再拍;而且脏快照只包含**真正会丢的用户输入字段**——把服务端回填的只读字段、内部状态也塞进快照,等于埋下无数个误报点。

## 三、样式:全局与作用域的两头坑

### 6. 全局工具类冲突

- **症状**:引入某个 UI 库或工具库之后,原本好好的布局到处错位,而且错得毫无规律。
- **根因**:项目里自定义了 `.flex` 这类**与框架/Tailwind 同名的全局工具类**,两边的同名类互相覆盖,谁赢取决于加载顺序,所以症状飘忽。
- **铁律**:全局工具类必须**带项目前缀**(示例:`.app-flex`),或者干脆不自定义与主流框架同名的类。已经踩了坑的,别指望调加载顺序续命——改名,一次痛,永久好。

### 7. scoped 样式写不进暗色

- **症状**:暗色模式下绝大部分界面正常,个别组件顽固地保持浅色,`html.dark .xxx` 的样式明明写了就是不生效。
- **根因**:那段暗色覆盖写在了 `<style scoped>` 里。scoped 会给选择器加属性隔离,导致 `html.dark` 这种挂在根元素上的选择器**够不着**组件内部元素。
- **铁律**:暗色覆盖统一写在**非 scoped 的样式块**里,选择器显式带上根类(`html.dark .my-component-class`),组件自己的 class 起得具体一点避免污染。

## 四、组件细节:一个日期选择器教的做人

### 8. 日期范围组件陷阱

- **症状**:用户选完日期范围点查询,接口报错或前端直接抛异常;而且不是每次都复现,跟操作路径有关。
- **根因**:日期范围组件在特定配置组合下会回抛 `null` 或与预期不符的值格式(比如期望 `'yyyy-MM-dd'` 字符串却拿到时间戳),下游代码没做防御,直接拼进请求。
- **铁律**:日期组件**统一封装一层**,项目里只允许用封装后的版本;封装里 `value-format` 显式指定、对 `null` 做兜底。UI 库组件的默认行为不是契约,显式声明的才是。

## 五、兼容与工程:设备和分支的账

### 9. 老安卓白屏

- **症状**:办公室所有电脑、所有新手机都正常,门店里几台旧安卓设备一打开管理端就白屏,连报错界面都没有。
- **根因**:构建产物用了太新的 JS 语法(可选链、新版 class 语法等),旧 WebView 解析阶段直接语法错误,一行业务代码都没跑到,所以白得干干净净。
- **铁律**:构建 target 锁 **es2015**(Vite 里 `build.target: 'es2015'`);发布前拿**在用的最老设备**真机过一遍关键页面。门店场景的设备更新周期远比你想象的长,「支持最近两个版本浏览器」这种默认假设在这里不成立。

### 10. 布局跳动(CLS)

- **症状**:打开列表页,内容先出来,顶部的汇总条(合计金额、总条数之类)晚一拍加载,整个列表「咯噔」往下跳一截,用户正要点的按钮跑位。
- **根因**:汇总条用 `v-if` 控制——数据没到就整个不渲染,不占位;数据到了突然插入,把下面的内容全部挤下去。
- **铁律**:会晚到的区块**常驻占位**:骨架屏或固定高度的空壳先立在那里,数据到了再填内容,绝不用 `v-if` 让它凭空出现。这是 CLS(Cumulative Layout Shift)最典型也最好治的一类。

### 11. 分支分叉恢复

- **症状**:把长期功能分支上的东西 cherry-pick 回主线,上线后功能「缺胳膊少腿」——主流程能跑,某个联动、某段样式、某个工具函数不见了。
- **根因**:窄挑提交必丢隐性依赖。长期分叉的分支上,一个功能往往散落在多个提交里,还依赖分叉期间的顺手小改(公共函数加参、样式微调),按提交挑永远挑不全。
- **铁律**:分叉超过一定时间后要搬功能,**整体 checkout 相关文件/目录**到目标分支(以文件的最终状态为准),不要按提交 cherry-pick;搬完把关键页面完整回归一遍,而不是只看新功能那一处。

## 踩坑与红线(速查卡)

上面 11 条的一句话版本,贴在工位上用:

| # | 症状 | 铁律 |
|---|---|---|
| 1 | 发版后 bug 复现不了 | 先让用户 Ctrl+Shift+R;资源带 hash,html 短缓存 |
| 2 | 部分用户永远旧页面 | html 边缘缓存 60 秒级 + 发版 purge;/api 排除缓存 |
| 3 | 切页丢筛选/闪重建 | router-view key 绑一级路由,不绑 fullPath |
| 4 | 切回列表数据是旧的 | onActivated 刷新 + 页码归位 |
| 5 | 没改也弹「未保存」 | 基线快照在数据加载后拍,只含会丢的字段 |
| 6 | 引 UI 库后布局乱 | 全局工具类加前缀,不与框架同名 |
| 7 | 暗色个别组件不变色 | 暗色覆盖写非 scoped 块,选择器带 html.dark |
| 8 | 选日期后查询报错 | 日期组件封装一层,value-format 显式指定 |
| 9 | 老安卓打开白屏 | build target 锁 es2015,最老真机过一遍 |
| 10 | 汇总条加载页面跳 | 常驻占位骨架,不用 v-if |
| 11 | cherry-pick 功能缺件 | 长期分叉整体 checkout 文件,不按提交挑 |
